# Critérios de Inclusão e Exclusão

## Critérios de Inclusão

| Código | Critério |
|--------|----------|
| I1 | Aborda prompt injection, jailbreak, ou vulnerabilidades relacionadas em LLMs |
| I2 | Discute sistemas agênticos, RAG, ou aplicações integradas com LLMs |
| I3 | Apresenta taxonomia, dataset/benchmark, estudo empírico, ou defesa avaliada |
| I4 | Publicado entre 2023 e 2026 |
| I5 | Texto completo disponível (PDF acessível) |
| I6 | Idioma: inglês ou português |

## Critérios de Exclusão

| Código | Critério | Exemplo |
|--------|----------|---------|
| E1 | Não é sobre LLM/GenAI | Segurança de redes tradicionais |
| E2 | É sobre LLM, mas não aborda segurança | Papers só sobre performance, fine-tuning |
| E3 | Conteúdo opinativo sem metodologia | Blog posts, tutoriais sem contribuição técnica |
| E4 | Duplicata (preprint + versão publicada) | Manter apenas a versão mais recente/completa |
| E5 | Texto completo não disponível | Paywall sem versão arXiv |
| E6 | Fora do escopo específico | Segurança em visão computacional sem relação com LLM |
| E7 | Idioma não suportado | Papers em chinês, russo, etc. |

## Como usar na planilha TRIAGEM

Na coluna "Motivo Exclusão", usar os códigos:
- Para incluir: deixar em branco ou "INCLUIR"
- Para excluir: usar E1, E2, E3, E4, E5, E6 ou E7
